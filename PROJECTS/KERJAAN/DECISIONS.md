## KERJAAN-D-001 — Project-specific source hierarchy

**Date:** 2026-06-18  
**Status:** ACTIVE

### Decision

Use this order for document work:

1. Ray's latest explicit instruction;
2. data typed by Ray;
3. designated spreadsheet;
4. target Word file;
5. base PDF or official source letter;
6. older examples;
7. historical chat context.

### Reason

Official document accuracy depends on using the correct source for each field and preventing old examples from overriding current instructions.

### Impact

- Personnel data comes from the designated list or spreadsheet.
- Base documents supply only relevant basis, purpose, date, place, wording, and signatory information.
- Material conflicts must be surfaced rather than silently resolved.

### Evidence or source

Explicit Ray project instructions and repeated KERJAAN workflows.

## KERJAAN-D-002 — Never invent official data

**Date:** 2026-06-18  
**Status:** ACTIVE

### Decision

Names, ranks, identifiers, positions, units, document numbers, dates, signatories, and official codes must come from a verified task source. Missing or ambiguous values use visible placeholders or are flagged for review.

### Reason

A plausible but incorrect value can create an invalid or misleading official document.

### Impact

- Preserve spelling, titles, abbreviations, and leading zeroes.
- Do not exchange data between people.
- Complete non-ambiguous sections while marking unresolved matches.

### Evidence or source

Explicit Ray project instructions and repeated reconciliation against spreadsheets.

## KERJAAN-D-003 — Preserve originals and document structure

**Date:** 2026-06-18  
**Status:** ACTIVE

### Decision

Modify only requested areas, preserve the supplied file structure, and save results separately unless Ray explicitly requests overwrite.

### Evidence or source

Explicit Ray project instructions and repeated requests to edit existing templates.

## KERJAAN-D-004 — Perkap Nomor 1 Tahun 2023 governs Polri Naskah Dinas work

**Date:** 2026-06-18  
**Status:** ACTIVE

### Decision

Use Peraturan Kepala Kepolisian Negara Republik Indonesia Nomor 1 Tahun 2023 as the governing reference for Polri Naskah Dinas structure and tata persuratan, together with the current task template and Ray's latest instruction.

### Reason

Each Polri document type has specific structure, numbering, address, attachment, signatory, margin, font, and distribution rules.

### Impact

- Classify the document type before drafting.
- Check the relevant Perkap section instead of applying one format to every document.
- Technical preparation is not the same as formal institutional approval.

### Evidence or source

User-supplied Perkap and Ray's explicit instruction to learn and apply it.

## KERJAAN-D-005 — Rendered-page verification is required

**Date:** 2026-06-18  
**Status:** ACTIVE

### Decision

Before describing a final Word or PDF as ready, inspect the rendered pages when layout matters.

### Reason

A saved file can still contain clipped text, table overflow, accidental blank pages, broken headers, or pagination defects.

### Impact

- Inspect all changed and continuation pages.
- Pair visual inspection with source-data verification.
- Report skipped visual checks and their impact.

### Evidence or source

Repeated KERJAAN workflows and Ray's evidence-over-claims standard.
