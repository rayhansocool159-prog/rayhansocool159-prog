# GitHub Ecosystem Learning and Profile Sync

**Owner:** Rayhan Haryo Pramudya

Use this checklist for an authorized review of the owner's GitHub repositories.

## Workflow

1. Verify the target account and profile repository.
2. Read the current global identity, working system, memory, ecosystem index, and template registry.
3. Inventory accessible owned repositories dynamically.
4. Inspect repository evidence beyond names and descriptions.
5. Classify each repository as `COMPLETE`, `SUBSTANTIAL`, `PARTIAL`, `MINIMAL`, or `BLOCKED` coverage.
6. Separate verified global lessons from project-specific, temporary, unverified, and non-public information.
7. Use calibrated labels: `IMPLEMENTED`, `VALIDATED`, `ACTIVELY DEVELOPING`, `DOCUMENTED`, or `EXPLORING`.
8. Prepare the smallest documentation-only update on a reviewable branch.
9. Check concurrent pull requests before editing overlapping files.
10. Read back every changed file and inspect the final diff.
11. Validate the canonical public owner name, Markdown links, claim accuracy, publication safety, changed-file scope, and mergeability.
12. Open a pull request and merge only when every check passes.

## Source priority

1. Latest explicit owner instruction.
2. Current repository files and state.
3. Active decisions and owner overrides.
4. Approved project documents.
5. Verified commits, tests, outputs, and artifacts.
6. Current-state handoffs.
7. Historical chats and assumptions.
