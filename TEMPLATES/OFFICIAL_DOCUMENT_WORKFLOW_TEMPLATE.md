## 6. Implementation rules

### Word

- preserve margins, fonts, spacing, indentation, numbering, tables, headers, footers, and signature areas;
- modify only requested content;
- add or remove rows only when necessary;
- remove obsolete sample data;
- save separately unless overwrite is authorized.

### Spreadsheet

- identify columns by header name;
- preserve formulas, sheets, formats, and leading zeroes;
- check blanks, duplicates, and mismatches;
- do not overwrite the source unless authorized.

### PDF

- inspect relevant pages visually when layout, signatures, stamps, tables, or images matter;
- use text extraction as supporting evidence only;
- ensure generated pages are readable and not clipped.

## 8. Acceptance criteria

- [ ] every requested record is present exactly once;
- [ ] no unintended sample or legacy record remains;
- [ ] names and identifiers match the designated source;
- [ ] dates, numbers, places, and signatories are consistent;
- [ ] numbering and grouping are correct;
- [ ] layout and continuation pages are intact;
- [ ] no clipped text, overflowing table, or accidental blank page;
- [ ] final file opens successfully;
- [ ] unresolved placeholders are visible and reported;
- [ ] source file remains preserved.

## 11. Promotion review

After completion, classify new learning as:

- **PROJECT-SPECIFIC** — keep in the project;
- **GLOBAL RAY MEMORY** — propose only if it reflects a durable owner preference;
- **REUSABLE TEMPLATE** — promote only if generic, stable, and free of private data.
